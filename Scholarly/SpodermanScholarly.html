Abstract

Spider Senses is a Web Extension that helps people with visual, auditive and motor disabilities to view, hear or read webpages that doesn't support this kind of access.

The user will be prompted to choose one of the options of the extension's option, depending on the user's disability.

The extension will also provide the user with the ability to make settings according to his/her preferences.

1. Introduction

On Spider Senses, users can select their disability, and the extension will make it easier for them to acces the selected webpage. Can't see all the colors? Can't hear well? Can't interact with your keyboard/mouse accordingly? Make it happen, with Spider Senses.

The visual aspect of the extension is kept as smooth and straightforward as possible, using a simple color scheme and an intuitive menu to cycle through.

2. Project Structure

Spider Senses extension aims to provide a smooth and enjoyable experience for the user.

Spider Senses has at it's core a manifest.json file (it's the main "ingredient") of an web extension. When the extension is clicked it opens with the Home page, and depending on your handicap you can select the most suitable option for you: Color Blind, Hearing Impairment or Motor Incapacity.

3. The visual aspect

The Spider Senses project interface consists of 4 pages (views), detailed below:

3.1. Home page

The home page is the landing page of the application, providing a list of 3 functions that it has for people with disabilities.

3.2. Color Blind Page

This is the page where people that are suffering of color blindness should go. This page would help them select the colors of the pages until they can see all of it. There is also provided a button to undo all the changes made, and one for returning to the Home page of the extension.

3.3. Hearing Impairment

With this option enabled, entering on an You Tube video will automatically activate the video's subtitles and it will raise the volume of the page, so the people with hearing impairment will have an easier time to hear what is actually going on. On this page there is also a button to undo all the changes made, and one for returning to the Home page of the extension.

3.4. Motor Incapacity

Choosing this option will open an interface that will access users camera and read on users face gestures, some js function will trigger for scrolling up and down a page or for increasing the volume etc. On this page there is also a button to undo all the changes made, and one for returning to the Home page of the extension.

4. Project progress

We started with the ideea that we want to make an extention that could really help people with disabilities (people with color blindness, hearing impairment or motor incapacity).

4.1. It all started with a manifest

The project began with the creation of the manifest.json file which is the fundation of an extension. In our case this extension works only for Google Chrome. In here we designed how the app will look in Google Extensions page. We put the mascot (little Spidey), the version, and a short description of the app.

4.2. Main HTML Page

The main page, or the home page, was the one that followed. We selected a background image that we think that is most visible for the color blind people and added the Spidey Senses mascot on it. We added then buttons that represent the actual choices of the incapacitated user: Spoder-Blind, Spoder-Deaf and Spoder-Motor. Each of the buttons has a on-click listener that erases all the content of the current page and then displays the selected button options.
